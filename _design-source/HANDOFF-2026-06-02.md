## 1. ここまでの確定事項（事業設計）

### 1.1 NOXA = 夜の街の OS（ナイトワーク総合プラットフォーム）
- 個人キャスト・ホストもオーナーも 1 アカウントで使う
- メイン業務モジュール 9 個: ①POS（決済なし・オーダー/伝票のみ）②売上管理(旧yorulog,実装済) ③席回し ④勤怠 ⑤給与 ⑥初回案内 ⑦送迎 ⑧在庫 ⑨オリシャン名刺発注

### 1.2 配信チャネルの建付け（2026-06-02 確定・最重要）
```
Web 版          = NOXA に完全統合（全機能=個人+店舗運営すべて）。ブランド NOXA
                  ↳ 実体は yorulog リポ（完成したアプリシェル (app)/ があるため）
よるログアプリ   = 売上管理専門のネイティブアプリ（個人キャスト/ホスト向け、②のみ）
  ├ iOS        = 別リポ yorulog-ios（SwiftUI）。絶対に触らない
  └ Android    = Capacitor WebView だと NOXA フルが見えてしまう → ネイティブ作り直し検討中（保留）
```
- **理由**: yorulog は元々「個人向け売上管理」。そこに店舗運営機能(POS等)を混ぜると役割破綻。配信チャネルで分離する。
- **規模で母体決定**: yorulog=69ページ+87コンポーネント（完成品）/ noxa=9ページ+3コンポーネント（LPのみ）。だから **yorulog リポを NOXA Web 本体にする**のが唯一現実的。逆移植はコンフリクト地獄。
- リポ統一（noxa リポの LP/account を yorulog に取り込む / GitHub リネーム yorulog→noxa-web / ドメイン noxa 化）は**別フェーズの大工事**。今はやらない。

### 1.3 のみシュギ最終形態 = C.ハイブリッド確定
独立ドメイン維持 + NOXA umbrella の /bars に内蔵(SSO連携)。Phase v1.3 で実施。

### 1.4 Firestore は個人/店舗を分離済み
